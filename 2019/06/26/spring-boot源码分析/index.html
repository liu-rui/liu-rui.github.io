spring boot 执行过程

  • 执行SpringApplication构造函数

- 赋值resourceLoader

- 通过输入参数赋值primarySources

- 计算出webApplicationType

- 通过spring-factories文件得到ApplicationContextInitializer的实现类，赋值initializers

  ApplicationContextInitializer一共添加了6个：

  org.springframework.boot.context.config.DelegatingApplicationContextInitializer

  org.springframework.boot.autoconfigure.SharedMetadataReaderFactoryContextInitializer

  org.springframework.boot.context.ContextIdApplicationContextInitializer

  org.springframework.boot.context.ConfigurationWarningsApplicationContextInitializer

  org.springframework.boot.web.context.ServerPortInfoApplicationContextInitializer

  org.springframework.boot.autoconfigure.logging.ConditionEvaluationReportLoggingListener

- 通过spring-factories文件得到ApplicationListener的实现类，赋值listeners

  ApplicationListener一共有10个：

  org.springframework.boot.context.config.ConfigFileApplicationListener

  org.springframework.boot.context.config.AnsiOutputApplicationListener

  org.springframework.boot.context.logging.LoggingApplicationListener

  org.springframework.boot.context.logging.ClasspathLoggingApplicationListener

  org.springframework.boot.autoconfigure.BackgroundPreinitializer

  org.springframework.boot.context.config.DelegatingApplicationListener

  org.springframework.boot.builder.ParentContextCloserApplicationListener

  org.springframework.boot.ClearCachesApplicationListener

  org.springframework.boot.context.FileEncodingApplicationListener

  org.springframework.boot.liquibase.LiquibaseServiceLocatorApplicationListener

- 通过主动抛出异常得到main主方法所在类，赋值给mainApplicationClass

  • 执行SpringApplication.run方法

- configureHeadlessProperty()  配置为headless模式

- 实例化对象SpringApplicationRunListeners，数据来源是通过spring-factories文件得到SpringApplicationRunListener的实现类,然后实例化得到的对象集合。目前只有一个EventPublishingRunListener；

- 调用SpringApplicationRunListeners对象的starting方法；广播应用程序启动事件(ApplicationStartingEvent)

- 根据输入参数args,构造ApplicationArguments

- configureIgnoreBeanInfo(environment)

- 创建可配置环境(ConfigurableEnvironment)对象

- 创建Banner

- 创建应用程序上下文(ConfigurableApplicationContext)

- 获取exceptionReporters

- 调用prepareContext方法，准备上下文

  设置上下文的环境

  调用postProcessApplicationContext(context)，前置处理上下文

  applyInitializers(context)，触发所有的ApplicationContextInitializer对象的initialize方法

  listeners.contextPrepared(context)，广播上下文已经准备好

  添加单例springApplicationArguments > applicationArguments类型

  添加单例springBootBanner > Banner

  设置允许bean定义按照名称覆盖，默认为禁止覆盖

  获取所有的源

  按照源列表加载bean

  ​    创建BeanDefinitionLoader

  ​    调用BeanDefinitionLoader.load方法

  ​        源可以是class,resource,package,字符串

  listeners.contextLoaded(context)，广播上下文已经导入

- 调用refreshContext方法，刷新上下文

  调用上下文的refresh方法

  注册上下文关闭回调

- afterRefresh，上下文刷新后调用，空的，子类实现

- 如果启动日志已经开启，打印启动耗时信息

- 调用SpringApplicationRunListeners对象的started方法；广播应用程序已经启动事件(ApplicationStartedEvent)

- callRunners(context, applicationArguments);

- 调用SpringApplicationRunListeners对象的running方法；广播应用程序已经就绪事件(ApplicationReadyEvent)

org.springframework.boot.SpringApplicationRunListener接口的方法签名：

  • void starting()
  • void environmentPrepared(ConfigurableEnvironment environment)
  • void contextPrepared(ConfigurableApplicationContext context)
  • void contextLoaded(ConfigurableApplicationContext context)
  • void started(ConfigurableApplicationContext context)
  • void running(ConfigurableApplicationContext context)
  • void failed(ConfigurableApplicationContext context, Throwable exception)
